# プロジェクトルール

## 基本方針
- Zenn向けの技術記事を執筆・管理するリポジトリ
- AIエージェントチームを活用して執筆〜レビューまで担当
- 人間（著者）はテーマ決定・専門知識提供・最終レビュー・公開判断を担う

## フォルダ構成ルール（全記事共通）

```
articles/{記事ID}/
├── index.md      # 記事本文（Zennへの公開ファイル）
├── outline.md    # 企画エージェントのアウトライン
└── research.md   # リサーチエージェントの調査結果
images/{記事ID}/  # 記事で使う画像
```

## 図解・画像ルール（全記事必須）

**すべての記事において、図解・画像を積極的に使うこと。**

### 使用すべき図解の種類
1. **Mermaidコード図** — Zennがネイティブ対応。フロー図・シーケンス図・構成図に使う
   ```mermaid
   graph LR ...
   ```
2. **Markdownテーブル** — スプレッドシート設計・比較表・データ構造の可視化に使う
3. **コードブロック付きコメント** — 処理の流れをインラインで説明
4. **ASCII構造図** — フォルダ階層・ディレクトリ構成の表示に使う（コードブロック内）

### 配置ガイドライン
- 各H2セクションに最低1つの図・表・コードブロックを入れる
- 「何が起きているか」を文章だけで説明しない → 必ず図で補完する
- フロー・アーキテクチャ・手順は Mermaid 図で可視化
- Before/After や比較はテーブルで整理

### 実際の画像ファイルを使う場合
- `images/{記事ID}/` に格納
- 参照方法: `![説明](../../images/ACC-002/screenshot.png)`

## 記事IDの命名規則
- `ACC-{番号}` : Autodesk ACC関連
- `PC-{番号}` : 点群処理関連
- `UMCP-{番号}` : Unity MCP関連
- `AI-{番号}` : AI/LLM活用関連
