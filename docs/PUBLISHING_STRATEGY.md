# Zenn パブリッシング戦略

## 1. ミッション

建設テック・3D技術・AI活用の分野で、実践的で価値のある技術記事を定期的に発信し、日本の技術コミュニティに貢献する。

## 2. ターゲット読者

- **建設テック系エンジニア**: BIM/CIM、ACC、点群処理を業務で扱う技術者
- **ゲーム/XR開発者**: Unity、MCPを使った開発者
- **AI/LLM活用に興味がある全エンジニア**: 業務へのAI導入を考えている技術者

## 3. コンテンツ4本柱

### 柱1: Autodesk ACC（Autodesk Construction Cloud）
- **API連携・自動化**: REST API、Webhook、SDKの使い方
- **運用ノウハウ**: プロジェクト管理、権限設定、ワークフロー構築
- **連携事例**: 他ツールとの統合、データフロー設計

### 柱2: 点群処理
- **取得技術**: LiDARスキャン、フォトグラメトリ
- **処理技術**: フィルタリング、位置合わせ、メッシュ化、セグメンテーション
- **可視化・活用**: Potree、Three.js、BIM連携、デジタルツイン

### 柱3: Unity MCP（Model Context Protocol）
- **実践的な活用例**: AIエージェントとUnityの連携
- **プロジェクト応用**: 実際の開発での利用パターン
- **Tips & トラブルシューティング**: ハマりやすいポイントと解決策

### 柱4: AI/LLM活用
- **建設分野でのAI活用**: 設計支援、品質管理、安全管理
- **開発ワークフロー改善**: Claude Code、GitHub Copilot等の活用
- **LLMアプリケーション開発**: RAG、エージェント、ツール連携

## 4. 投稿スケジュール

**頻度**: 週1回（毎週公開）

### ローテーション方針

4つのカテゴリを均等にローテーションする：

| 週 | カテゴリ |
|----|---------|
| 第1週 | Autodesk ACC |
| 第2週 | 点群処理 |
| 第3週 | Unity MCP |
| 第4週 | AI/LLM活用 |

→ 各カテゴリ月1回のペースで継続的にコンテンツが蓄積される。

### 柔軟性ルール
- 時事的なトピック（新機能リリース、カンファレンス等）がある場合はローテーションを崩してOK
- ネタの準備状況に応じて前後の入れ替えは自由
- 連載モノは連続週での投稿も可

## 5. 記事タイプの使い分け

| タイプ | 使いどころ | 目安文字数 |
|--------|-----------|-----------|
| **チュートリアル** | 手順を追って実装できるハンズオン | 3,000〜6,000字 |
| **解説記事** | 概念・仕組み・アーキテクチャの説明 | 2,000〜4,000字 |
| **Tips集** | 短い実践的なノウハウ | 1,000〜2,000字 |
| **事例紹介** | 実プロジェクトでの適用事例 | 2,000〜3,000字 |

## 6. 品質基準

### 記事に必ず含めるもの
- [ ] 明確なタイトル（何が学べるかわかる）
- [ ] 対象読者レベルの明記
- [ ] 実際に動くコード or 具体的な手順
- [ ] 参考リンク・公式ドキュメントへの誘導
- [ ] まとめ（記事のポイント整理）

### やらないこと
- 公式ドキュメントの単なる翻訳
- 検証していない情報の掲載
- 古いバージョンの情報を最新のように書く

## 7. SEO・発見性の工夫

- **タイトル**: 検索されやすいキーワードを含める（例: 「ACC APIでファイル一覧を取得する方法」）
- **タグ**: Zennのタグを適切に設定（最大5個）
- **トピック**: tech（技術記事）を基本にidea（アイデア）も活用
- **シリーズ化**: 同カテゴリの記事はシリーズとしてリンクを張る

## 8. KPI（目標指標）

### 短期（3ヶ月）
- 記事数: 12本以上
- 4カテゴリ各3本以上

### 中期（6ヶ月）
- 記事数: 24本以上
- Like数の合計: 50以上
- フォロワー: 増加傾向を確認

### 長期（1年）
- 記事数: 48本以上
- 各カテゴリでの認知獲得
- Book（連載形式）を1つ以上公開
