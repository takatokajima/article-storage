# AIエージェントチーム運用ガイド

## 概要

記事の企画から公開まで、AIエージェントチームが各フェーズを担当する。
人間（著者）はディレクション・最終判断・専門知識の提供を行い、AIが下書き・構成・レビューを分担する。

## エージェント役割定義

### 1. 企画エージェント（Planner）

**役割**: 記事テーマの提案とアウトライン作成

**実行タイミング**: 毎月初め or 記事ネタが必要なとき

**プロンプト例**:
```
あなたは技術記事の企画担当です。以下の条件で次に書くべき記事のテーマを3つ提案してください。

- カテゴリ: [ACC / 点群 / Unity MCP / AI・LLM]
- ターゲット: 幅広い技術者
- 既存記事: [既存の記事タイトル一覧を添付]
- 最近のトレンド: [該当カテゴリの最近の動向]

各提案について以下を含めてください:
1. タイトル案
2. 想定読者レベル（初級/中級/上級）
3. 記事タイプ（チュートリアル/解説/Tips/事例）
4. アウトライン（H2レベルの見出し）
5. 差別化ポイント（他の記事にない価値）
```

### 2. リサーチエージェント（Researcher）

**役割**: 記事に必要な技術情報の調査と整理

**実行タイミング**: アウトライン決定後、執筆前

**プロンプト例**:
```
以下のアウトラインに基づいて、記事執筆に必要な技術情報を調査・整理してください。

タイトル: [記事タイトル]
アウトライン: [H2見出し一覧]

各セクションについて以下をまとめてください:
1. 公式ドキュメントの該当箇所とURL
2. 関連するコード例やAPI仕様
3. 注意点・よくある落とし穴
4. 参考になる既存記事（差別化のために把握）
```

### 3. 執筆エージェント（Writer）

**役割**: 記事本文の下書き作成

**実行タイミング**: リサーチ完了後

**プロンプト例**:
```
以下の情報をもとに、Zenn記事の下書きを作成してください。

タイトル: [記事タイトル]
タイプ: [チュートリアル/解説/Tips/事例]
アウトライン: [H2見出し一覧]
リサーチ結果: [リサーチエージェントの出力]

ルール:
- Zenn記事のフロントマター（title, emoji, type, topics, published）を含める
- コードブロックには言語指定をつける
- 読者が実際に手を動かせる具体性を持たせる
- 各セクションの冒頭に「このセクションでわかること」を簡潔に記載
- 結論は先に書く（PREP法）
```

### 4. レビューエージェント（Reviewer）

**役割**: 下書きの品質チェックとフィードバック

**実行タイミング**: 下書き完了後

**プロンプト例**:
```
以下のZenn記事の下書きをレビューしてください。

[下書き全文]

以下の観点でチェックし、改善提案を箇条書きで出してください:

## 技術的正確性
- コード例は正しく動くか
- APIの仕様は最新か
- 用語は正確か

## 読みやすさ
- 構成は論理的か
- 説明の飛躍はないか
- 初学者にも伝わるか

## Zenn最適化
- タイトルは検索されやすいか
- タグは適切か
- 適切な長さか（短すぎ/長すぎないか）

## 差別化
- 他の既存記事と比べて独自の価値はあるか
- 著者の経験や知見が反映されているか
```

### 5. 編集エージェント（Editor）

**役割**: レビュー結果を反映した最終稿の仕上げ

**実行タイミング**: レビュー完了後

**プロンプト例**:
```
以下の下書きにレビューフィードバックを反映して、公開可能な最終稿を仕上げてください。

下書き: [下書き全文]
レビュー結果: [レビューエージェントの出力]
著者の追加指示: [あれば]

仕上げルール:
- フロントマターのpublishedはfalseのまま（人間が確認後にtrueにする）
- 誤字脱字を修正
- 文体を統一（です・ます調）
- コードブロックのインデントを整える
```

## ワークフロー

```
[月初] 企画エージェント → テーマ3案提案
          ↓
[著者] テーマ選定 & 追加情報提供
          ↓
[執筆週] リサーチエージェント → 技術情報整理
          ↓
         執筆エージェント → 下書き作成
          ↓
         レビューエージェント → 品質チェック
          ↓
         編集エージェント → 最終稿仕上げ
          ↓
[著者] 最終確認 → published: true → マージ → 公開
```

## 実行方法（Claude Codeでの運用）

### 1セッションで記事を書く場合

```
あなたはZenn記事のAIエージェントチームです。
以下の5つの役割を順番に実行して、記事を完成させてください。

1. 企画: テーマ「[テーマ]」のアウトライン作成
2. リサーチ: 必要な技術情報の調査
3. 執筆: 下書きの作成
4. レビュー: 品質チェック
5. 編集: 最終稿の仕上げ

カテゴリ: [ACC / 点群 / Unity MCP / AI・LLM]
追加コンテキスト: [著者の知見や経験を記載]
```

### 月初の企画セッション

```
今月のZenn記事の企画を行います。
docs/EDITORIAL_CALENDAR.mdを参照し、
次に書くべき記事のテーマを各カテゴリ1つずつ、計4つ提案してください。
既存の記事一覧: articles/ ディレクトリの記事を参照
```

## 著者の役割（人間がやること）

1. **テーマの最終決定**: AIの提案から選ぶ or 自分で指定
2. **専門知識の提供**: 実務経験、ハマりポイント、独自の知見を口頭で伝える
3. **技術的な検証**: コード例が実際に動くか確認
4. **最終レビュー**: published: trueにする前の最終チェック
5. **公開判断**: マージ & 公開の決定

## Tips

- 記事のネタは思いついたら `docs/EDITORIAL_CALENDAR.md` の「ネタストック」に追記しておく
- 1記事あたりの作業時間目安: AIとの協業で **1〜2時間**
- 完璧を目指さず、まず公開してから改善するスタイルを推奨
- シリーズものは最初にシリーズ全体の構成を決めてから着手
